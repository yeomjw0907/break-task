# TaskBrick Landing IA

## Goal

The landing page should not sell TaskBrick as a fun todo app.

It should convert users around one sharp promise:

> See how much of your day became real work, then close the day with proof.

Primary landing objective:

- get visitors to understand the category in under 10 seconds
- create immediate self-recognition
- push them to try the product or join a waitlist

## Audience

Primary audience:

- freelancers
- solo founders
- operators
- marketers
- planners

Secondary audience:

- productivity-minded desk workers
- focus-challenged users who need structure without shame

## Messaging Principle

Lead with:

- truthful workday visibility
- daily closure
- real focus versus desk time

Do not lead with:

- gamification
- coins / combos / scoreboards
- "AI productivity"
- "fun todo app"

Those can appear later as supporting details.

## Page Structure

### 1. Hero

Purpose:

- explain the product in one line
- create immediate pattern break
- establish category

Headline options:

- `오래 일한 게 아니라, 얼마나 진짜로 밀었는지 보여줍니다.`
- `출근부터 퇴근까지, 당신의 실제 업무를 기록합니다.`
- `오늘 뭐 했는지 남기고, 성취감 있게 하루를 닫으세요.`

Subcopy:

- `TaskBrick은 할 일 관리 앱이 아닙니다.`
- `실집중 시간, 전환 횟수, 완료 기록, 퇴근 리포트를 한 번에 보여주는 개인 업무 운영체제입니다.`

Primary CTA:

- `오늘부터 기록하기`

Secondary CTA:

- `퇴근 리포트 보기`

Hero visual:

- one strong product shot
- focus on `clock in / focus / clock out report`
- show `work time vs focus time` clearly

Do not:

- show too many cards
- explain every feature in the hero
- overemphasize score effects

### 2. Problem Recognition

Purpose:

- make the user feel seen

Section title:

- `이런 날이 자주 있나요?`

Bullets:

- `오래 일했는데 실제로 뭘 했는지 모르겠다`
- `할 일은 많은데 자꾸 다른 일로 샌다`
- `퇴근할 때 성취감보다 찜찜함이 남는다`
- `노션, 캘린더, 메모, 투두가 다 따로 논다`

Close with:

- `TaskBrick은 이 하루의 흐름을 하나로 묶습니다.`

### 3. How It Works

Purpose:

- make the workflow feel simple and believable

Three-step flow:

1. `출근`
   - 오늘 핵심 3개와 근무 예산을 잡는다
2. `집중`
   - 작업을 시작하고 전환과 복귀를 기록한다
3. `퇴근`
   - 근무 시간과 실집중 시간을 비교하고 하루를 닫는다

Microcopy:

- `가볍게 시작하고`
- `실제로 밀고`
- `분명하게 마감합니다`

### 4. Truth Metrics

Purpose:

- show why this is different from a normal planner

Section title:

- `당신의 하루를 이렇게 봅니다`

Metrics to feature:

- `근무 시간`
- `실집중 시간`
- `집중 비율`
- `전환 횟수`
- `끼어들기`
- `시간대 집중 패턴`

Interpretation line:

- `바빴던 하루`와 `실제로 밀린 하루`는 다를 수 있습니다.

### 5. Product Preview

Purpose:

- turn abstract value into product understanding

Show 3-5 annotated panels:

- `Start Day`
- `Today Board`
- `Focus Console`
- `Clock-out Report`
- `Weekly Review`

Each preview should have:

- one short label
- one short explanation
- one visible user outcome

Example:

- `Clock-out Report`
- `오늘 실제로 얼마나 일했는지 확인하고, 내일로 넘길 일을 정리합니다.`

### 6. Proof / Why It Sticks

Purpose:

- explain repeat usage

Content:

- `출근`이 하루의 시작 루틴이 된다
- `퇴근 리포트`가 하루의 마감 루틴이 된다
- `점수와 콤보`는 실행을 밀어주는 보상으로만 작동한다

Possible proof blocks:

- fake but realistic example report
- "before / after" day comparison
- first user quotes later when available

### 7. Mid-page CTA

Purpose:

- catch convinced users before the bottom

CTA:

- `내 하루 진짜 보기`

Support line:

- `오늘부터 출근과 퇴근을 기록해보세요.`

### 8. Trust / Positioning Guardrails

Purpose:

- reduce category confusion

Section title:

- `TaskBrick은 이런 제품입니다`

Checklist:

- `프로젝트 관리 툴이 아닙니다`
- `감시 툴이 아닙니다`
- `과한 RPG 앱이 아닙니다`
- `직장인의 하루를 실행하고 닫는 도구입니다`

### 9. Final CTA

Purpose:

- restate the promise cleanly

Headline:

- `오늘을 그냥 보내지 말고, 기록하고 닫으세요.`

CTA options:

- `오늘부터 기록하기`
- `퇴근 리포트 써보기`

## Conversion Strategy

### Primary conversion

- try the app

### Secondary conversion

- join waitlist
- request early access

### CTA repetition

- hero
- mid-page
- final section

## Visual Direction

- desktop-first
- dark + warm neutral base
- amber only as reward and primary action accent
- product screenshots over illustration-heavy marketing
- professional first, arcade second

## What To Avoid

- selling the product as a "gamified todo app"
- too many feature bullets near the top
- long founder story before the product value is clear
- abstract lifestyle copy with no concrete work pain
- loud animations in the marketing page

## Recommended First Draft

Order:

1. Hero
2. Problem Recognition
3. How It Works
4. Truth Metrics
5. Product Preview
6. Why It Sticks
7. Mid-page CTA
8. Trust / Not another app
9. Final CTA

If only one message must survive, it should be:

`TaskBrick shows how much of your day became real work, then helps you close the day with clarity.`
